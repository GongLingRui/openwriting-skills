# 分析引擎规则

## 一、进度健康度评估模型

### 单任务健康度评分

对每项任务按以下维度打分（满分100）：

| 维度 | 权重 | 评分规则 |
|------|------|---------|
| 计划-执行吻合度 | 40% | 本周进展内容与上周计划的匹配程度 |
| 进展实质性 | 25% | 进展描述是否包含具体成果/数据/节点 |
| 问题密度 | 20% | 是否存在未解决问题，问题严重程度 |
| 连续性 | 15% | 是否连续多周有实质进展 |

### 计划-执行吻合度判定规则

比较「上周的本周计划」与「本周的上周进展」：

- **高度吻合（90-100分）**：进展内容基本覆盖计划中的所有要点
- **部分完成（60-89分）**：进展内容覆盖计划中的部分要点，其余推迟
- **偏离计划（30-59分）**：进展内容与计划不一致，但有其他实质进展
- **基本无进展（0-29分）**：无实质进展描述，或仅有"持续跟进"等空泛表述

### 板块健康度

板块健康度 = 板块内所有任务健康度的加权平均值
- 重点任务权重 = 2
- 常规任务权重 = 1

### 状态灯判定

| 状态 | 条件 |
|------|------|
| 🟢 正常 | 板块健康度 ≥ 70 |
| 🟡 关注 | 板块健康度 50-69，或存在1项以上重点任务无进展 |
| 🔴 警告 | 板块健康度 < 50，或存在重点任务连续2周以上无进展 |

## 二、异常检测规则

### 2.1 任务停滞检测

**判定逻辑**：连续N周的"进展"描述无实质变化

- 判定方式：
  1. 进展字段为空
  2. 进展内容与上周完全相同（文本相似度 > 90%）
  3. 进展内容仅包含空泛表述（如"持续跟进""继续推进""暂无进展"等）

- 预警级别：
  - 连续1周无进展 + 重点任务 → 🟡 提示
  - 连续2周无进展 → 🟡 关注
  - 连续3周及以上无进展 → 🔴 严重

### 2.2 问题升级检测

**判定逻辑**：同一"存在问题/需协调事项"跨周重复出现

- 判定方式：取连续两周的"存在问题"字段，比较核心内容（忽略细微措辞变化）
- 如果同一问题连续出现2周 → 标记为"持续未解决"
- 如果同一问题连续出现3周及以上 → 标记为"长期阻塞"，建议升级处理

### 2.3 计划漂移检测

**判定逻辑**：本周计划内容与上周计划不一致，且无对应的进展说明

- 判定方式：
  1. 对比上周"工作计划"与本周"工作计划"
  2. 如果计划内容发生显著变化（非正常推进导致的更新）
  3. 且上周计划中的内容在本周进展中未体现
  → 标记为"计划漂移"

### 2.4 空泛表述识别词库

以下表述视为无实质内容，在评分和异常检测中降权处理：

```
持续跟进
继续推进
暂无进展
暂无
无
后续跟进
持续跟踪
按计划推进
正常推进中
```

## 三、团队绩效评估维度

### 3.1 任务完成率

```
完成率 = 本周有实质进展的任务数 / 团队承担的总任务数 × 100%
```

### 3.2 计划一致性

```
一致性 = 上周计划在本周得到执行的任务数 / 上周有计划的任务数 × 100%
```

### 3.3 问题响应速度

```
评估问题从首次出现到消失所经历的周数：
- 1周内解决 → 优秀
- 2周内解决 → 良好
- 3周及以上 → 需关注
```

### 3.4 重点任务推进率

```
推进率 = 有进展的重点任务数 / 团队承担的重点任务数 × 100%
```

### 综合评级

| 等级 | 条件 |
|------|------|
| A (优秀) | 完成率≥80% 且 一致性≥80% 且 重点推进率≥90% |
| B (良好) | 完成率≥60% 且 一致性≥60% 且 重点推进率≥70% |
| C (一般) | 完成率≥40% 或 重点推进率≥50% |
| D (需改进) | 完成率<40% 或 重点推进率<50% |

## 四、向上汇报要素提炼规则

### 4.1 成果提炼

从周进展数据中提取以下类型的成果：

1. **完成类成果**：含有"完成""已完成""已上报""已提交"等关键词
2. **突破类成果**：含有"首次""创新""突破""上线"等关键词
3. **量化成果**：含有具体数字、百分比、数量的进展

### 4.2 问题提炼

分三个层次组织问题：
1. **体制机制类**：涉及制度设计、流程优化的深层问题
2. **协调推进类**：涉及跨部门/跨团队配合的问题
3. **技术能力类**：涉及系统开发、数据治理的技术问题

### 4.3 语言转化规则

周报中的口语化、碎片化描述 → 汇报材料中的规范化表述

| 周报原文风格 | 转化为汇报材料风格 |
|------------|-----------------|
| "这周做完了XXX" | "已完成XXX相关工作" |
| "XXX还没搞定" | "XXX工作正在有序推进中" |
| "跟XXX沟通了一下" | "已与XXX进行专题对接" |
| "问题还是那个问题" | "相关问题仍待进一步协调解决" |
| "催了好几次" | "已多次督促推进" |
| "方案改了好几版" | "经多轮优化完善" |

## 五、跨周趋势分析

### 5.1 月度趋势指标

按月统计以下指标，用于月度管理报告的趋势分析：

- 各板块活跃任务数变化
- 新增问题数与解决问题数
- 重点任务推进率变化曲线
- 各团队完成率变化

### 5.2 风险预判逻辑

结合关键节点日历（milestone-calendar.md），评估：

1. 距离下一个关键节点还有多少周
2. 当前推进速度能否按期完成
3. 如果存在风险，提前多少周发出预警

预警提前量：
- 一般节点：提前2周预警
- 总部报送节点：提前3周预警
- 风委会/党委会/董事会上会：提前4周预警
